# 🐉 DnD Chronicles

Сайт-помощник для игры в Dungeons & Dragons с друзьями: бестиарий, книга заклинаний, справочник, кампании, конструктор персонажей и листы персонажей.

## Структура проекта

```
dnd-chronicles/
├── index.html              # разметка страниц
├── css/
│   └── style.css           # все стили
└── js/
    ├── data/                # контент — правь только здесь
    │   ├── monsters.js      # бестиарий
    │   ├── spells.js        # заклинания
    │   ├── reference.js     # классы/расы/оружие/броня/снаряжение
    │   └── campaigns.js     # стартовые кампании
    ├── navigation.js         # переключение страниц
    ├── bestiary.js            # логика бестиария
    ├── spells.js               # логика книги заклинаний
    ├── reference.js            # логика справочника
    ├── campaigns.js            # логика кампаний
    ├── character-creator.js   # пошаговый конструктор персонажа
    ├── character-sheet.js     # список и лист персонажей
    ├── utils.js                # модальные окна, уведомления
    └── init.js                 # запуск всего при загрузке страницы
```

## Локальный запуск

Открыть `index.html` двойным кликом не получится — браузер блокирует загрузку отдельных `.js`-файлов через `file://`. Нужен локальный сервер:

```bash
# из папки проекта
python3 -m http.server 8000
```

Затем открыть `http://localhost:8000` в браузере.

Либо в VS Code: установить расширение **Live Server** и нажать "Go Live".

## Как добавить контент

- **Новый монстр** → `js/data/monsters.js`
- **Новое заклинание** → `js/data/spells.js`
- **Новый класс/раса/оружие** → `js/data/reference.js`
- **Кампания** (стартовая, до создания через интерфейс) → `js/data/campaigns.js`

Логику менять не нужно — только сами данные в этих файлах.

## Хранение данных

Персонажи, созданные через сайт, сохраняются в `localStorage` браузера. Это значит:
- данные не пропадают при перезагрузке страницы;
- но они привязаны к конкретному браузеру/устройству — у каждого друга будет свой список персонажей.

## Публикация на GitHub Pages

1. Создать новый репозиторий на [github.com](https://github.com/new)
2. Залить туда содержимое этой папки (см. команды ниже)
3. В репозитории: **Settings → Pages → Source** → выбрать ветку `main` и папку `/ (root)`
4. Через минуту сайт будет доступен по адресу `https://<твой-логин>.github.io/<название-репозитория>/`

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<твой-логин>/<название-репозитория>.git
git push -u origin main
```

После любых изменений в файлах:

```bash
git add .
git commit -m "Описание изменений"
git push
```

Сайт на GitHub Pages обновится автоматически через 1-2 минуты после пуша.
